## The Problem

Crypto and markets research still means a dozen tabs — CoinGecko, explorers, news, charts — and no single place to ask a question in plain English and get analytics back. Agenticly was built to sit that assistant on both phone and web, with subscriptions that work on either surface.

## What I Built

Agenticly is a Turborepo product: an **Expo / React Native** mobile app and a **React + Vite** web app against Firebase for auth/data, with a **FastAPI + LangGraph** backend for streaming research chat.

- Natural-language queries across crypto (CoinGecko), on-chain (BigQuery), and stocks (Alpha Vantage), with interactive Plotly charts.
- Stock recommendation / analyst-ranking flows alongside freeform research chat (SSE streaming).
- Credits plus Pro/Enterprise tiers — **RevenueCat** for mobile store billing, **Stripe** for web — normalized into one entitlement model in Firebase so access doesn’t depend on which surface you paid on.

I owned architecture, both clients, backend, payments, and store submission. Live on [Play Store](https://play.google.com/store/apps/details?id=computer.star.agenticly) and [agenticly.app](https://agenticly.app).

## Major Features

- **NL research assistant** — ask in English; get structured analytics and charts, not a SQL console.
- **Multi-source data** — crypto, on-chain, and equities behind one chat surface.
- **Cross-platform billing** — RevenueCat + Stripe into a single entitlement truth.
- **Streaming answers** — SSE chat so long research replies feel live.
- **Team / settings / payment history** — product surfaces beyond the demo chat box.

## Key Challenges

Keeping subscription state coherent across RevenueCat and Stripe — different webhooks, renewal semantics, and failure modes — meant the Firebase layer had to normalize both into one entitlement record. Shipping mobile and web as separate clients (not a forced shared UI kit) kept launch speed high for a solo build.

## Results

Agenticly is live on Play Store and web with subscription revenue on both channels, built and shipped solo from architecture through launch.

## Stack Decisions

Expo + Vite React were chosen for fast dual-platform shipping rather than maximizing code reuse. Firebase kept auth/data overhead low; FastAPI/LangGraph own the agent loop. RevenueCat absorbs App Store / Play receipt complexity that would otherwise be hand-rolled.
