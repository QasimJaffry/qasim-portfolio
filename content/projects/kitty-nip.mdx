## The Problem

Kitty Nip is a location-based social app that had already found an audience, but needed an engineer to take over full ownership — including a backend that had to keep working reliably as the user base grew past its early stage.

## What I Built

I took over full product ownership of Kitty Nip, a React Native app backed by a Node.js and PostgreSQL backend, with Firebase handling auth and real-time features. That ownership included the mobile client, backend services, and the operational work of keeping the app stable and maintained as it scaled.

## Key Challenges

Taking over an existing codebase mid-flight meant understanding and stabilizing the backend before making changes, since the app already had real users depending on it. Scaling the location-based features as downloads grew required attention to database query patterns and indexing in PostgreSQL to keep response times consistent under load.

## Results

Kitty Nip has passed 10,000+ downloads on the Play Store under my ownership, with the backend included in that scope of responsibility.

## Stack Decisions

The existing React Native and Node.js/PostgreSQL stack was retained rather than rewritten — a location-based social app with a real, growing user base doesn't justify a rewrite risk. Firebase was kept in place for the real-time and auth pieces it was already handling well.
